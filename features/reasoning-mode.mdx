Reasoning Mode combines Deepseek's advanced reasoning capabilities with Gemini's search tools to provide transparent, strategic analysis of complex queries.

## What Makes It Different

Unlike other search modes that directly answer your query, Reasoning Mode takes a two-stage approach:

<Steps>
  <Step title="Reasoning Stage" icon="brain">
    **Deepseek Reasoner** analyzes your query and creates a search strategy
    
    - Identifies information needs
    - Defines search approach
    - Plans explanation structure
    - Shows its thought process in real-time
  </Step>

  <Step title="Search Stage" icon="magnifying-glass">
    **Gemini 2.0 Flash** executes the strategy and delivers results
    
    - Performs ≥10 targeted searches
    - Synthesizes information
    - Structures comprehensive response
    - Includes images and related questions
  </Step>
</Steps>

<Info>
  The key innovation is transparency: you see **how** the AI approaches your query before seeing the answer.
</Info>

## How It Works

### Architecture

<Tabs>
  <Tab title="Overview">
    ```mermaid
    graph LR
      A[User Query] --> B[Deepseek Reasoner]
      B --> C[Reasoning Strategy]
      C --> D[Gemini 2.0 Flash]
      D --> E[Google Search Tool]
      E --> F[Final Answer]
    ```

    **Stage 1: Reasoning**
    - Model: Deepseek Reasoner (configurable)
    - Input: User query + language preference
    - Output: Search strategy (streamed)
    - Duration: ~3-5 seconds

    **Stage 2: Search & Synthesis**
    - Model: Gemini 2.0 Flash
    - Input: Query + reasoning strategy
    - Output: Comprehensive answer with sources
    - Duration: ~5-10 seconds
  </Tab>

  <Tab title="API Flow">
    ### 1. Reasoning Request

    ```typescript
    // GET /api/reasoning?q=your_query&language=English

    const response = await deepseek.chat.completions.create({
      model: REASON_MODEL || "deepseek-reasoner",
      messages: [
        {
          role: "system",
          content: `You are a helpful reasoner, acting as the **initial explorer** 
                   for a Search Agent. Your role is to **define the search strategy**, 
                   not to find the answer itself.`
        },
        {
          role: "user",
          content: `The user query is: ${query}. Please provide your search 
                   strategy and explanation plan. Your answer should start with 
                   <think> and end with </think>`
        }
      ],
      stream: true  // Real-time streaming
    });
    ```

    ### 2. Streaming Response

    ```typescript
    // Client receives reasoning chunks in real-time
    res.setHeader('Content-Type', 'text/event-stream');
    
    for await (const chunk of reasoningResponse) {
      const content = chunk.choices?.[0]?.delta.content;
      if (content) {
        reasoning += content;
        res.write(`data: ${JSON.stringify({ content })}\n\n`);
      }
    }
    
    // Send complete reasoning
    res.write(`data: ${JSON.stringify({ complete: true, reasoning })}\n\n`);
    ```

    ### 3. Search Request

    ```typescript
    // POST /api/search with reasoning context

    const searchPrompt = `Following reasoning analysis:
    ${reasoning}
    
    to search and answer Query: ${query}`;

    const result = await chat.sendMessage(searchPrompt);
    ```
  </Tab>

## Reasoning System Prompt

The Deepseek model receives specific instructions to act as a "search strategist":

```
You are a helpful reasoner, acting as the **initial explorer** for a Search Agent.
Your role is to **define the search strategy**, not to find the answer itself.

Your task is to think like a search expert and determine:

1. **Information Needs:** What specific types of information, facts, or data are
   absolutely necessary to fully understand and answer the user's query? Think
   about the *categories* of information we need to search for.

2. **Explanation Strategy:** Once we have all the necessary information, how
   should we structure our explanation to clearly and comprehensively answer
   the user's query? Outline the *key points* or *logical steps* of the
   explanation.

Remember, your output should be a **search and explanation plan**, not the
answer. Focus on *how* we will search and *how* we will explain, rather than
*what* the answer is.
```

<Warning>
  The reasoning model is explicitly instructed NOT to answer the query directly, only to plan the search strategy.
</Warning>

## Model Configuration

### Deepseek Reasoner Settings

<ParamField path="model" type="string" default="deepseek-reasoner">
  The reasoning model (configurable via `REASON_MODEL` environment variable)
</ParamField>

<ParamField path="stream" type="boolean" default={true}>
  Enables real-time streaming of reasoning process
</ParamField>

<ParamField path="messages" type="array" required>
  System prompt + user query requesting search strategy
</ParamField>

### Gemini Search Settings

<ParamField path="temperature" type="number" default={1.0}>
  Balanced creativity for comprehensive responses
</ParamField>

<ParamField path="topP" type="number" default={0.95}>
  Top 95% probability mass for response diversity
</ParamField>

<ParamField path="topK" type="number" default={40}>
  Consider top 40 tokens for generation
</ParamField>

<ParamField path="maxOutputTokens" type="number" default={65536}>
  Full output capacity for detailed responses
</ParamField>

<ParamField path="systemInstruction" type="string" required>
  Instructs model to follow reasoning strategy and perform ≥10 searches
</ParamField>

## Benefits of Two-Stage Reasoning

### 1. Transparency

```typescript

```

You can verify that the AI:
- Understood your question correctly
- Identified relevant information needs
- Planned an appropriate search strategy
- Structured a logical explanation

```
Without Reasoning:
"universal basic income" → Generic search → Generic answer

With Reasoning:
"UBI pilot programs results" +
"UBI inflation concerns economic theory" +
"UBI labor market effects" +
... → Targeted searches → Comprehensive answer
```

The reasoning plan becomes an outline for the final answer:

```
Reasoning Plan:          →    Final Response:
1. Definition                  ## What is UBI?
2. Economic Effects            ## Economic Implications
3. Empirical Evidence          ## Research Findings
4. Implementation              ## Practical Considerations
5. Conclusion                  ## Summary
```

### 4. Educational Value

Users learn:
- How to break down complex questions
- What information is relevant
- How to structure research
- Critical thinking approaches

## API Usage

<CodeGroup>
```bash GET Reasoning Only
# Stream the reasoning process
curl "https://api.omnisearches.com/api/reasoning?q=your+query&language=English"
```

```bash POST Full Search
# Use reasoning mode for search
curl -X POST https://api.omnisearches.com/api/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": "your query",
    "mode": "reasoning",
    "language": "English"
  }'
```

```typescript Client Integration
// 1. Fetch reasoning
const reasoningStream = await fetch(
  `/api/reasoning?q=${encodeURIComponent(query)}&language=English`
);

const reader = reasoningStream.body.getReader();
let reasoning = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const text = new TextDecoder().decode(value);
  const lines = text.split('\n').filter(l => l.startsWith('data: '));
  
  for (const line of lines) {
    const data = JSON.parse(line.slice(6));
    if (data.content) {
      reasoning += data.content;
      // Display reasoning in real-time
      updateReasoningDisplay(reasoning);
    }
  }
}

// 2. Use reasoning for search
const searchResponse = await fetch('/api/search', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    query,
    mode: 'reasoning',
    reasoning,  // Include reasoning context
    language: 'English'
  })
});

const results = await searchResponse.json();
```
</CodeGroup>

## Performance Considerations

<Warning>
  Reasoning Mode is the slowest mode due to the two-stage process. Use it when transparency and strategic analysis are more important than speed.
</Warning>

### Timing Breakdown

```
Total Time: ~8-15 seconds
├─ Reasoning Stage: 3-5s
│  ├─ Deepseek processing: 2-4s
│  └─ Streaming to client: 1s
└─ Search Stage: 5-10s
   ├─ 10+ web searches: 3-6s
   ├─ Content synthesis: 1-2s
   ├─ Image fetching: 1-2s (parallel)
   └─ Response formatting: <1s
```

### Optimization Tips

<AccordionGroup>
  <Accordion title="Client-Side">
    - Display reasoning as it streams (don't wait for completion)
    - Show loading indicators during search stage
    - Cache reasoning for follow-up questions in same session
  </Accordion>

  <Accordion title="Server-Side">
    - Use streaming for reasoning (reduces perceived latency)
    - Fetch images concurrently with search results
    - Consider caching common reasoning patterns
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Reasoning stream stops unexpectedly">
    Check that:
    - `REASON_MODEL_API_KEY` is valid
    - API endpoint is accessible
    - Network connection is stable
    - Client properly handles SSE events
  </Accordion>

  <Accordion title="Search doesn't follow reasoning">
    Ensure:
    - Reasoning text is properly passed to search endpoint
    - Gemini system prompt includes instruction to follow reasoning
    - Reasoning output is well-formed (starts with `<think>`, ends with `</think>`)
  </Accordion>

  <Accordion title="Performance is too slow">
    Consider:
    - Using Default or Exhaustive mode for faster results
    - Reducing `maxOutputTokens` in reasoning stage
    - Implementing server-side caching
    - Using a faster reasoning model
  </Accordion>
</AccordionGroup>
